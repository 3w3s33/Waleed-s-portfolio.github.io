Generic

This code defines a React application with two main components: Inventory and App. The Inventory component is responsible for managing an inventory of products, allowing the user to adjust the quantity of items in two categories: Electronics and Home Appliances. The App component serves as the root of the application, rendering the Inventory component along with a welcome message. Within the Inventory component, two state variables are created using React's useState hook. The first state variable, electronics, holds an array of objects, each representing a product in the electronics category with a name and an initial quantity set to 0. The second state variable, homeAppliances, similarly holds an array of objects for products in the home appliances category, each also starting with a quantity of 0. These state variables enable the component to track and update the quantities of the products. The component defines a function called handleQuantityChange to handle changes in the quantity of a product. This function takes three parameters: category, index, and delta. The category determines whether the product belongs to the electronics or home appliances category. The index represents the position of the product in the respective array, and the delta specifies the amount to change the quantity by, either increasing or decreasing the quantity. The function updates the quantity of the selected product and ensures that the quantity does not go below zero. The updated state is then set using either setElectronics or setHomeAppliances depending on the category. The component renders a list of products from both the electronics and home appliances categories.
Each product is displayed with its name, current quantity, and two buttons to increment or decrement the quantity. When the user clicks the "+" button, the quantity increases, and when they click the "-" button, the quantity decreases. The handleQuantityChange function is called when either button is clicked to adjust the quantity. The Inventory component also includes a "Submit" button that triggers the handleSubmit function when clicked. This function simply shows an alert confirming that the inventory has been updated successfully. In a more complex implementation, this step could involve sending the updated quantities to a backend server for storage. The App component serves as the root component of the application. It displays a welcome message and a prompt asking the user to select the desired products. The Inventory component is then rendered within the App component, allowing users to interact with the inventory. The entire application is rendered to the DOM using the ReactDOM.render() method, which inserts the App component into the HTML element with the ID root. This makes the application visible to the user on the web page.
